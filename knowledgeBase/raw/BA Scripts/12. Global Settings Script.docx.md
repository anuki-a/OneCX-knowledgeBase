# **Accessing Global Settings** 

Under the ‘Configuration’ section in the left menu, click on ‘Global Settings’ option to access the 'Global Settings’ page. The global settings are grouped into four main categories:

# **1\. Basic Settings**

Here, you can manage the client information such as like the Client Display Name (used across UI, email headers, and browser tabs), and configure email fields:

* Email From Display Name  
* Email Reply Address  
* Email From Email  
* Email Sender Address  
* Client Admin Email

Other important fields include:

* Timeout Duration: Default is 90 minutes of inactivity.  
* Text Notification Toggle: Enables SMS opt-in across Lobby, Kiosk, and Widget.  
* Default Location Address: Displayed when location sharing is disabled (Widget only)  
* Client Logo Upload: Used across Kiosk and Web Portal. The system will auto-adjust the logos sizes and display it in all platforms and devices.

# **2\. Authentication Settings**

You can configure:

* Username/Password authentication or ADFS single sign-on and identity provider settings  
* Password expiration policies (e.g., 90 days by default)  
* Two-factor authentication

When using ‘Single Sign On’ authentication, a new domain field has also been introduced — this allows configuration of multiple domains per client, with the ability to mark one as the default. This ensures the right user association during Just-In-Time provisioning which can be configured within the ‘Just-In-Time Provisioning’ section.

# **3\. Appointment Settings**

This includes controlling the:

* Float Travel Pad Time by adding a buffer time for float staff to travel to different branches for servicing appointments  
* Allow Widget Appointments via Portal and  
* Enablement of Confirmation Emails being sent from Assigned Employees

# **4\. Lobby Settings**

Here client admins can define:

* The default tab (Lobby Queue, Appointment Status, Central Views) displayed when users open the Lobby module.  
* Alert notification chime which will be used when alerts are triggered through the application.  
* Staff Status Display Text   
* Lobby specific options like mandating comments, enabling reference numbers for products/services, interaction alerts and browser pop up. Moreover, the staff preference selections and impersonation-related settings can also be configured.  
* Visibility of optional fields in Lobby like Account Number, DOB, Email, and Custom Fields.  
* Day To Regenerate Previous Month's Data for ‘Analytics and Scheduler File Export’ can also be set.

Finally, you can click on “Save” to update changes made within the ‘Global Settings’ page. Click ‘Close’ to minimize the expanded section

And that wraps up our walkthrough of the Global Settings module in OneCX. 

These tools give administrators full control over product access, security, branding, and customer interaction settings across all locations.
