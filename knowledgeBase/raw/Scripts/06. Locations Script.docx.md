# Locations Script

**Intro – Overview**  
Welcome to the OneCX platform — your centralized solution for seamless customer engagement. In this video, we'll walk through the Locations module in detail. This module allows client administrators to configure, manage, and maintain branch-specific settings such as hours of operation, services, holidays, and lobby behaviors.

**Accessing the Locations Module**  
To access the Locations module, navigate to the Locations & Services section and click on ‘Locations’.   
 You will be directed to a grid that displays all configured locations.

**Locations Grid Overview**  
The grid view provides a high-level summary of all created locations. Columns include:

* Location Code  
* Location Name and Display Name  
* Type \- Organization, Area, Region or Branch  
* Parent Location  
* License \- Appointments and/or Lobby Tracker  
* Action \- Edit icon to update details and lock/unlock icon to activate/deactivate locations)

Filter options are available by active status, location type, locations, and inclusion of sub-locations.  
You can also use the search bar to filter locations.   
Click the ‘Add Location’ button to create a new one or use the edit icon to modify an existing location.

**Configuring a Location \- Task Based UI Overview**  
The location add/edit screen uses a task-based UI organized into multiple collapsible sections:

* Basic Settings  
* Operating Hours  
* Holidays  
* Products & Services  
* Appointment Settings  
* Lobby Settings

We’ll walk through each section in detail below.

1. ### **Basic Settings Section** 

### This section is divided into two key subsections: **Basic Information** and **Contact Information**.

#### **Basic Information**

This captures the core identification and configuration values required to set up the location within OneCX.

* **Location Code** **\-** A unique identifier for the location within the system. This must be distinct and is typically used in backend mappings and reports.  
* **Location Name** **\-** This is the official name of the location as it appears in administrative contexts throughout the OneCX web portal.  
* **Location Display Name** **\-** This is the user-facing name shown across front-end modules like kiosk and appointment booking widget interfaces. It can be more descriptive or client friendly.  
* **Type \-** Specifies whether the location is an Organization, Region, Area or a Branch. Branch types represent physical service centers, while Region types are typically used for grouping multiple branches together.  
* **Time Zone \-** Determines the time context for the location.  
* **Parent \-** Identifies the parent node in the hierarchy for structuring sub-locations.  
* **Active Checkbox \-** When checked, the location is considered active and will be available for user interaction throughout the application. Inactive locations will be hidden from most dropdowns and cannot be used for appointments or lobby check-ins.

#### **Contact Information**

This section captures the location's physical address and communication details.

* **Address 1** \- Primary address line for the location (e.g., street address or PO box).  
* **Address 2 \-** Secondary address line, used for additional location descriptors such as floor or suite numbers.  
* **City** \- City where the location is physically situated.  
* **State** \- The U.S. state (or equivalent region) corresponding to the address.  
* **ZIP Code \-** Postal code for mailing purposes.  
* **Country**\- Pre-filled or selectable country value (defaults to *United States*).  
* **Phone Number / Phone Ext \-** Main contact number for the location and optional extension if applicable.  
* **Email \-** A designated email address of the location used for a client or internal communication.  
* **Latitude / Longitude \-** These are auto-generated fields which are updated automatically when address information is completed.

When saving a new location for the first time, we have two options as ‘Save’ and ‘Save and Continue’

* Clicking on ‘Save’ navigates back to the location grid. The location appears in the main grid.  
* Clicking on ‘Save and Continue’ retains the same view allowing you to edit other settings after saving the location.

When closing the section, click “Close” to minimize the expanded section  

2. ### **Operating Hours**

This section helps define the standard working hours for the selected location. The layout is organized by the open status, days of the week, starting from Sunday through Saturday, and allows you to define the hours during which the branch will be open for operations.

For each day, you can add one or more time slots using the “+ Add” button next to it. This is especially useful if the location takes a break during the middle of the day — for instance, operating from 9 AM to 12 PM, and then again from 1 PM to 5 PM.

To add a time range, simply click the “Add” button, set the Start Time and End Time, and you’re done. If a time slot needs to be removed, you can easily click the trash icon next to that row.

If a particular day is considered a non-working day — for example, Sunday — you can simply uncheck the ‘Open’ status checkbox of the related row.

The values configured here will determine when the branch is open for both walk-in lobby interactions and appointment scheduling, unless overridden by any special holiday settings in the next section.

This setup serves as the default operating schedule for the location and plays a key role in determining what slots are available to users during check-ins or online bookings.

3. ### **Holidays**

Holidays cannot be manually added through this interface. Instead, holidays are provisioned into the system through the ‘Data Import/Export’ module. Once they are uploaded, based on the opt in/out settings provisioned, the holidays will be applied at the location level.

If the location you're viewing is a branch, then you’ll see all holidays that have been opted-in specifically for that branch. However, if you're looking at a higher-level location, such as an organization, area or region, the system will automatically show any holidays that have been opted in for at least one of its child branches. This gives you full visibility into all holidays affecting locations under that parent.

The provisioned holidays will be displayed in a grid view. Columns include:

* Opt In – whether the holiday is opted in/out for the related location.  
* Holiday Name   
* Date   
* Start Time and End Time of the holiday   
* All Day  
* Blackout  
* Status – Inherited or Overridden from the parent settings.  
* Action – to edit the holiday settings. 

Filter options are available by the Opt In Status, Inherited Status and Year.  
Clicking the edit icon will display the ‘Edit Holiday’ view where you can opt in/out that holiday for the current branch and modify its settings. If you’re accessing a higher-level location, this lets you centrally manage how that holiday cascades down to underlying branches.

4. ### **Products & Services**

This section displays all the products and services that are available in the system — and shows you which ones are opted in/out for the currently selected location.

By default, the list will show only the services opted for this specific branch. If the location you’re viewing is a higher-level location, such as a region, area or organization, then you’ll see only the services that have been opted in for at least one of its child branches.

Each row in the grid includes:

* The Service Code, Name and Display Name  
* Its associated Category and Type

Filter options are available by the Opt In Status, Product Category, Type and Language.  
While this section allows you to view the opted-in services, any changes to the opt-in status are typically handled through the ‘Products/Services’ module — not from this location-specific view.

The ‘Set Display Order’ hyperlink will be displayed for ‘Branch’ type locations at the top right of this section. This will navigate you to the ‘Set Display Order’ page, where you can adjust how the services are presented across multiple channels: Lobby Check-In, Kiosk and in the Lobby Wrap-Up view.

You can reorder the services using drag-and-drop, either globally or by category — depending on whether the ‘Show Category’ option is enabled in the kiosk and global settings for the lobby.

* If “Show Category” is enabled, the display order set per category will be displayed for the location.  
* If “Show Category” is disabled, the system will use the global display order defined for the location.

The changes you make here are specific to this branch — you also have the flexibility to override global display orders and, when needed, apply this customized order across other branches as well.

5. ### **Appointment Settings**

This section controls how the selected location handles appointment scheduling and related preferences.

Let’s walk through each of the options available in this section:

* **Use Appointments \-** Start by enabling this checkbox if you want this location to support appointments.  
* **Future Availability Window (In Days) \-** This setting defines how far into the future users can book appointments.   
* **Appointment Cut-Off Window (In Days) \-** This is the cut-off range for editing or cancelling upcoming appointments.   
* **Summary Emails Notification Time \-** This defines the time of day when daily appointment summary emails should be triggered.  
* **Reminder Email and Text Message Notification (Before Appointment Time – In Hours)** \- These settings control when reminder emails and text messages are sent.  
* **Allowed Lead Time for Booking Appointments \-** This defines the minimum period required before someone can book an appointment.   
* **Concurrency Mode** \- This controls how the system handles multiple appointments, exceptions, and resources in the same time slot. In this case, you're allowed to set a Max Number of Appointments per slot — like 2\. So up to two appointments can occur at the same time.  
* **Float Assignment Priority Mode \-** This setting defines which users should be considered first when assigning appointments — especially in cases where both home and float employees are available.  
* **Change Appointment Status Automatically at Configured Auto-Wrap Up Time \-** This setting defines how the system should automatically update appointment statuses if they haven't been manually wrapped up. You can choose between the ‘Completed’ and ‘No Show’ status options.  
* **Allow Widget User to Request Staff** \- Enable this if you want users booking appointments through the widget to request a preferred staff member. If disabled, staff will be assigned automatically.  
* **Show Location in Widget** \- This option controls whether this branch appears in the public-facing appointment widget.   
* **Use Events** \- If your organization uses event scheduling, this setting enables that behavior for this specific location.

This section works in conjunction with the availability settings defined earlier and ensures that only valid appointment slots and formats are exposed to both internal staff and end-users.

6. ### **Lobby Settings**

This section includes two settings that control how remote check-ins are handled for this specific branch.

* **Allow Remote Check-In \-** When this option is enabled, customers will be able to check into the branch remotely for an already booked appointment, even before physically arriving.   
*  **Restrict Remote Check-In Outside of Operating Hours \-** If enabled, the system will block any remote check-in attempts that fall outside the branch's configured availability hours — even if remote check-in is generally allowed.

This helps prevent early check-ins before the branch opens, or after hours when staff may not be available to assist.

**Saving a Section** – Click on “Save” to update changes done in the ‘Basic Settings’, ‘Operating Hours’, ‘Holidays’, ‘Products/Services’, ‘Appointment Settings’ and ‘Lobby Settings’ sections. 

**Closing a Section** – Click “Close” to minimize the expanded section

And that wraps up our walkthrough of the Locations module in OneCX. From configuring core location details to managing holidays, products/services, availability, appointments and lobby settings — this module gives client administrators the flexibility to define how each branch operates and serves its customers.
