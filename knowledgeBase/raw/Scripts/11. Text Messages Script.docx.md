**Accessing Text Messages**  
To begin, navigate to the ‘Configuration’ section on the left panel, and click ‘Text Messages’.

**Lobby Text Configuration**

At the top of the page, you’ll see the ‘Lobby Text Configuration’ section.  
This is where you define automated text messages that will be sent to customers based on actions in the lobby.

* First, choose your Language — currently only ‘English’ is supported hence the dropdown value is pre-selected and disabled.  
* Then select the Text Message Type — you’ll see these options:  
  * Walk-in  
  * Appointment  
  * Remote Check-In  
  * Queue

These message types are pre-configured from the backend.

* You can choose whether a message type is enabled using the Opt In checkbox.  
* Saving will apply this setting globally across all locations. 

**Editing Message Content**

* Next, under ‘Message Content’ section, you can customize the text for each message type by clicking on the ‘Edit’ icon.  
  Drag and drop chips such as ‘CustomerFirstName’ or ‘LocationDisplayName’ to dynamically insert values. The chips will be displayed specific to the text message type.

**Reset to Default**  
If you want to restore the default template, simply click the ‘Reset to Default’ button. Please note, this action cannot be reversed once applied.

**How Automated Messages Work**  
Here’s how each message type works:

* Walk-in — triggered when a walk-in customer checks in.  
* Appointment — triggered when an appointment customer checks in.  
* Remote Check-In — triggered when a customer performs a remote check-in.  
* Queue — triggered based on queue position.  
  For example: If set to 1, customers will receive a message when they are next in line.

**Text On Demand — Sending Ad-hoc Messages**  
When a customer is checked in and has opted in to receive text messages, you’ll see a message icon in the ‘Action’ column of the related record within the lobby queue.

Staff members can click this icon to open a message window — where they can type a custom message (up to 350 characters) and click the ‘Send’ button. This allows staff to communicate ad-hoc with customers waiting in the lobby.

**Viewing Text On Demand History**  
All ad-hoc messages sent using this feature are automatically recorded in the ‘Text On Demand History’ section — just below ‘Lobby Text Configuration’ section in the ‘Text Messages’ page.

Here you can:

* Filter messages by date range  
* Select one or more locations  
* View messages sent by specific staff members  
* Perform a keyword search

All matching results will display in the grid below.

And that’s a quick walkthrough of the ‘Text Messages’ module in OneCX. These tools are designed to help you manage both automated and on-demand communication with your customers — right from your lobby.
