* Click on ‘Permission Groups’ under the ‘User Management’ section.

## **Overview & Walkthrough**

The Permission Groups module in OneCX plays a central role in managing both user access and system notifications. This page is used by both new and migrating clients:

* If you are a new client coming into OneCX, you can begin by creating permission groups by clicking the ‘Add’ button at the top of the Permission Groups page. A search bar is also available for filtering existing groups.  
* If you are an existing client migrating from the Appointment Concierge and/or Lobby Tracker platforms, you will find previously defined groups already available in OneCX. During migration, the permissions from these platforms are automatically mapped to equivalent permissions in OneCX, avoiding the need for rework.

The grid on this page displays the following columns:

* Permission Group Name  
* Notification Group – A checkbox indicating if the group is used solely for notifications.  
* Actions – Includes an edit icon for modifying the group.

**Creating or Editing a Group**    
Whether you're creating a new group via the “Add” button or editing an existing one through the edit icon, the layout is the same.

1. **Group Name** – Enter a name for the group in the “Permission Group Name” field. This is a required field.   
     
2. **Notification Group Checkbox** – This checkbox determines the purpose of the group:  
* If checked, the group is treated as a Notification Group.  
  * All permission checkboxes will be disabled.  
  * The group can be assigned to a user for alert and notification targeting.  
  * A user can belong to multiple notification groups.  
      
* If unchecked, the group is treated as a Permission Group.  
  * You can configure feature-level access by selecting checkboxes under different permission sections, including:  
    * Administrative Permission  
    * Lobby Permissions  
    * Analytics Permissions  
    * Appointments Permissions  
  * You can select individual permissions, “Select All” in each category, or use the master checkbox to select all within a section.  
    A user can be assigned to only one permission group. This group controls which parts of OneCX they can access.  
      
3. **Saving a Group** – Click “Save” to create or update the group. Once saved:  
* The group appears in the main grid.  
* It becomes available for assignment in user profiles and, if applicable, in the alert configuration module.  
    
4. **Cancelling the changes** – Click “Cancel” to discard any unsaved changes.  
* You will be redirected back to the Permission Groups page.

And that’s a quick walkthrough of the Permission Groups module in OneCX. These tools are designed to give you complete control over how your system is configured and presented to users.
