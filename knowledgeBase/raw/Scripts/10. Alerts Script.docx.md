# Alerts Script

# **Intro — Overview**

Welcome to the OneCX platform — your unified suite for streamlined branch operations and configuration management. In this video, we'll walk you through the Alerts feature — a real-time notification feature designed to alert staff members based on lobby activity and predefined check-in criteria.

There are two types of alerts in OneCX:

* **Global Alerts:** These alerts are configured by administrators and can be assigned to individual users and notification groups (which may include multiple users). Global Alerts are typically used to manage broader operational notifications across the organization.  
* **My Alerts:** These are personalized alerts configured by individual users for their own needs. Users can define the specific conditions under which they wish to be notified, allowing for a tailored experience.

Let’s now explore how each of these works.

# **Global Alerts**

1. # **Accessing Global Alerts Configuration**

To access the ‘Global Alerts’ module, go to the ‘Configuration’ module and click on ‘Global Alerts’ from the side menu.  
\[Visual: Navigate to Configuration → Global Alerts\]

You will be directed to a grid that displays all configured alerts.  

2. **Alerts Grid Overview**

The grid view provides a high-level summary of all created alerts. Columns include:

* **Notify** – The user or group who will receive the configured alert when it is triggered.  
* **Alert Type** – The medium through which the alert will be triggered: System Alert, Email, or Text Message (SMS).  
* **Phone Number** – The phone number to which SMS alerts will be triggered, if applicable.  
* **When** – Specifies when the alert should be triggered: immediately after check-in or after the wait time exceeds a defined threshold.  
* **Request Type** – Defines the requested user, group, product, or service (P/S) in the lobby record that the alert is tied to.  
* **Notify Offline** – Indicates whether the alert should be triggered even if the notifier is offline.  
* **Action** – Provides options to edit or delete an existing alert.

\[Visual: Highlight grid columns and action buttons\]

3. # **Creating Global Alerts**

To add a new global alert, click on the the ‘Add Alert’ button at the top right of the page. This will enable inline editing, and a new row will appear at the top of the grid to enter the alert details. You’ll also see the ‘Save’ and ‘Cancel’ buttons under the Action column to either confirm or discard the new alert.

\[Visual: Show the Add Alert button and click it\]

## **Choose Who to Notify**

You have multiple options to decide who gets alerted:

* **Individuals** – Select users based on your home location and its sub-locations.  
* **Groups** – Select predefined notification groups created for alert purposes.  
* **Skilled Users** – Set alerts for skilled users proficient in providing a specific product or service.  
* **Email Lists** – Manually enter one or more email addresses if you want to alert someone outside the system.

## **Choose How Alerts Are Sent**

You can determine the delivery method from the below options:

* **System Alert** – In-app notifications.  
* **Email** – Alerts sent to email inboxes.  
* **SMS** – Messages sent to users’ phone numbers specified in their profiles, with the option to include additional numbers.  
* **All** – Use every method to ensure the message is delivered.

## **Set When Alerts Are Triggered** 

You can control the timing of your alerts, so that the alert is sent:

* **Immediately** – As soon as a customer checks into the lobby.  
* **After a Wait Time** – Only when the lobby wait time for a customer exceeds the limit you set.

## **Filter Alerts Based On Lobby Requests**

You can customize alerts to trigger based on what’s being requested:

* **All Requests at Location** – Sends alerts for any request.  
* **No Requests** – Useful if you want to catch lobby check-ins with no users, groups or services requested.  
* **Custom Requests** – Pick exactly which users, groups, or P/S entries requested should trigger the alert.

## **Notify Offline Option**

Alerts can be sent even when users are not signed into the portal. This feature is disabled by default and not applicable for email list notifiers.

\[Visual: Show the options available for adding a global alert\]

4. # **Editing Global Alerts**

To modify an existing alert, users can click the ‘Edit’ icon under the ‘Action’ column. This action makes the selected row editable directly within the grid.

Once in edit mode:

* All fields within the row become active, allowing updates to any part of the configuration (e.g., Notify, Via, When, Request(s), etc.).  
* The Save icon is enabled once any changes are made. Clicking this saves the updated alert and returns the row to its non-editable state.  
* The Cancel icon discards any unsaved changes and reverts the row to its original state.

\[Visual: Show the Edit feature\].

5. # **Deleting Global Alerts**

To remove an alert, users can click the ‘Delete’ icon located in the ‘Action’ column of the corresponding row.

**My Alerts**

In addition to Global Alerts, users can also configure their own alerts. These alerts are specific to an individual user and allow for personalized notifications, without affecting the alert settings for the rest of the organization.

The ‘My Alerts’ page is accessible through the header panel of the OneCX application. Click your username and select ‘My Alerts’ from the dropdown menu. You will be directed to a grid that displays all configured ‘my alerts’ along with two action buttons to ‘Edit’ or ‘Delete’ an alert. 

Any ‘Global Alerts’ assigned to the logged in user will be visible in this grid as well.

\[Visual: Navigate to My Alerts and show the grid\]

The layout and functionality for adding, editing, and deleting alerts in the ‘My Alerts’ page follows the same behavior as described for the ‘Global Alerts’ module.

And that wraps up our walkthrough of the ‘Alerts’ feature in OneCX.

With the ability to configure ‘Global Alerts’ for groups and ‘My Alerts’ for individual preferences, you can ensure timely, targeted notifications that improve responsiveness and streamline branch operations.
