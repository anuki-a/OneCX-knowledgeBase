# Impersonation & Transfers Script

# **Intro — Overview**

Welcome to the OneCX platform — your unified suite for streamlined branch operations and configuration management. In this video, we will explore the Impersonation & Transfers feature — where you can impersonate another staff member in the Lobby or transfer the Lobby assistance to another staff member.

\[Visual: Show the OneCX homepage and navigate to Configuration \> Global Settings.\]

1. # **Enabling Impersonation for the Client**

To begin using the impersonation functionality, we must ensure that impersonation is enabled for the client.   
First, go to Configuration \> Global Settings, then scroll down to ‘Lobby Settings’ section. Here, enable the checkbox "Enable User Impersonation" for the client.

\[Visual: Navigate through the menu as described and show the checkbox being enabled.\]

2. # **Enabling Impersonation for Users**

Once you’ve enabled and saved the impersonation setting for the client at the global level, the next step is to configure it at the user level. Navigate to the ‘Users’ module, select the specific user, and go to their ‘Lobby Settings’.

\[Visual: Show navigating to Users module → selecting a user → opening Lobby Settings.\]

Here, you will see two additional settings:

* Enable User Impersonation  
* Show User for Impersonation

These options are visible only if impersonation is enabled for the client in the global settings.

\[Visual: Highlight both checkboxes and their labels.\]

* Select "Enable User Impersonation" if you want this user to be able to impersonate other users.  
* Select "Show User for Impersonation" if you want this user to be impersonated by others.

\[Visual: Demonstrate checking each option with appropriate tooltips or info icons.\]

3. # **Impersonating Users in the Lobby**

Once a user with the “Enable User Impersonation” setting turned on signs into the lobby, they can begin assisting visitors on behalf of other users through two main methods:

* Instant Assist (Impersonate)  
  Once a visitor checks in, an Instant Assist (Impersonate) button will appear in the Visitor Check-In pop-up.  
  Clicking on this allows the signed-in user to immediately assist the visitor by impersonating another staff member.  
  \[Visual: Show the Visitor Check-In screen with the Instant Assist (Impersonate) button.\]  
    
* Impersonate Assist from the Lobby Queue  
  Alternatively, users can assist visitors who are already in the Lobby Queue.  
  Click on the Impersonate Assist icon in the relevant lobby record to initiate the process.  
  \[Visual: Show Lobby Queue with impersonate icons next to visitor entries.\]

In either case, the user will be prompted with a pop-up showing a list of staff members they can impersonate.

This list includes users who:

* Have the “Show User for Impersonation” setting enabled, and  
* Are currently signed out of the relevant location’s lobby.

\[Visual: Show pop-up list with eligible users for impersonation.\]

Once a user is selected, the staff member can start assisting the visitor as that impersonated user.

You can refer to the Lobby Training Video for more details on how to perform a standard assist.

\[Visual: Insert an on-screen note linking to the Lobby training video.\]

A user can simultaneously impersonate and assist up to 25 visitors in the lobby.

4. # **Transferring an Impersonation to Other User(s)**

If you're currently impersonating a user and want to transfer a visitor to another user to continue the assist — while still maintaining the original impersonation — you can do so using the Transfer Assists feature.

\[Visual: Show the user assisting a visitor while impersonating someone else.\]

To begin, click the ‘Transfer Assists’ button located at the bottom of the left navigation panel.

\[Visual: Highlight and click the Transfer Assists button in the UI.\]

This opens the Transfer Assists (Impersonation) pop-up.

Here, you have two options:

* Transfer All: Move all ongoing impersonated assists to a single eligible user.  
* Transfer Individually: Select specific impersonated assists and assign each to a different eligible user.

\[Visual: Display the pop-up with both options clearly shown.\]

Once the transfers are made:

* The new users taking over the impersonated assists will be notified automatically, and  
* The assist will continue under the same impersonated identity — ensuring a seamless experience for both staff and visitors.

\[Visual: Show notifications sent and updated assist status under impersonation.\]

And that wraps up our walkthrough of the Impersonation & Transfers module in OneCX.

This feature empowers your staff to collaborate more efficiently, ensure seamless visitor experiences, and manage handoffs across teams — all while maintaining control and visibility.
